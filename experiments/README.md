# experiments/

Measurable experiments and spikes live here — one directory or file per
experiment. An experiment is a small, bounded attempt to prove or learn
something, tied to a Goal and Expected evidence (see the Experiment issue
template).

Record results as hard evidence in `docs/findings-log.md`. Keep experiments
small and reversible. Throwaway spikes can be deleted once their findings are
logged; the finding outlives the code.
